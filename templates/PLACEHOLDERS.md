# Placeholder ทั้งหมดในระบบ ConsDocx

อ้างอิงสำหรับตอนทำเอกสารเพิ่ม (เอกสาร 6, 7, ...) — เอา placeholder ที่มีอยู่แล้วไปใช้ซ้ำได้เลย
ไม่ต้องคิดชื่อใหม่ ถ้าความหมายตรงกับที่มีอยู่แล้ว

รูปแบบ placeholder ในไฟล์ `.docx`: พิมพ์ข้อความ `[ชื่อ_placeholder]` ธรรมดาลงในเอกสาร (มีวงเล็บเหลี่ยมครอบ)
โค้ดฝั่ง JS จะค้นแทนที่ข้อความนี้ด้วยค่าจริงตอนสร้างไฟล์

## Placeholder ที่ใช้ร่วมกันหลายเอกสาร

| Placeholder | ความหมาย | ใช้ในเอกสาร | ค่ามาจาก |
|---|---|---|---|
| `[ชื่อโครงการ]` | ชื่อโครงการก่อสร้าง | 1, 2, 3, 4, 5 | ช่อง "ชื่อโครงการ" หน้า main (`projectName`) |
| `[เดือน]` | เดือนที่ออกเอกสาร | 1, 2, 3, 4, 5 | ช่อง "เดือน/ปี" หน้า main (`g_month`) |
| `[ปี]` | ปี พ.ศ. ที่ออกเอกสาร | 1, 2, 3, 4, 5 | ช่อง "เดือน/ปี" หน้า main (`g_year`) |
| `[จำนวนเงินตัวเลข]` | งบประมาณ ตัวเลขพร้อม comma | 3, 4, 5 | ช่อง "งบประมาณ" หน้า main (`g_moneyNum`) ผ่าน `formatComma()` |
| `[จำนวนเงินตัวอักษร]` | งบประมาณ แปลงเป็นตัวอักษรไทย | 3, 4, 5 | ช่อง "งบประมาณ" หน้า main (`g_moneyNum`) ผ่าน `bahtText()` |

## เรื่องเดิม / ข้อเท็จจริง (เอกสาร 1 และ 5 เท่านั้น)

| Placeholder | ความหมาย | ค่ามาจาก |
|---|---|---|
| `[PM1]` `[PM2]` `[PM3]` `[PM4]` | ย่อหน้า "เรื่องเดิม" — แต่ละอันอยู่คนละย่อหน้าในเอกสาร | ช่อง "เรื่องเดิม" หน้า main (`g_priorMatter`) |
| `[ข้อเท็จจริง]` | ข้อความ "ข้อเท็จจริง" | ช่อง "ข้อเท็จจริง" หน้า main (`g_facts`) |

**กลไกพิเศษของ PM1-4:** ถ้าผู้ใช้พิมพ์ข้อความเองในช่อง "เรื่องเดิม" ข้อความนั้นจะถูกใส่ใน `[PM1]`
อย่างเดียว ส่วนย่อหน้า `[PM2]`-`[PM4]` (ที่ปกติมีข้อความตัวอย่าง/default อยู่) จะถูก**ลบทั้งย่อหน้าทิ้ง**
ไปเลย ไม่ใช่แค่เว้นว่าง — ถ้าผู้ใช้ไม่พิมพ์อะไรเลย จะใช้ข้อความ default 4 ท่อน (คงที่ในโค้ด `DEFAULT_PM_1`
ถึง `DEFAULT_PM_4`) ใส่ใน 4 ย่อหน้าตามเดิม

เหตุที่ต้องแยกเป็น 4 placeholder แทนที่จะเป็นก้อนเดียว: เทมเพลตต้นฉบับเขียนย่อหน้า "เรื่องเดิม" เป็น
4 บรรทัดแยกกัน (แต่ละบรรทัดอ้างอิงกฎหมาย/มติ/งบประมาณคนละเรื่อง) เวลาผู้ใช้กรอกเอง มักมีแค่เนื้อหาเดียว
เลยยุบเหลือย่อหน้าเดียว (PM1) แล้วลบ 3 ย่อหน้าที่เหลือทิ้ง

ถ้าจะทำ placeholder แบบ "ลบทั้งย่อหน้าถ้าไม่มีค่า" ในเอกสารใหม่ ให้ส่งค่า `null` (ไม่ใช่ `""`) เข้า
`fieldsMap` ตอนเรียก `buildOneDocx()` — โค้ดจะลบ `<w:p>...</w:p>` ทั้งก้อนที่มี placeholder นั้นทิ้งให้

## ฟังก์ชันช่วยแปลงค่า (ใช้ซ้ำได้ทุกเอกสารใหม่)

อยู่ใน `forms/main.html` และซ้ำแบบเดียวกันในแต่ละไฟล์ฟอร์มของเอกสารนั้นๆ:

- `formatComma(numStr)` — ใส่ comma คั่นหลักพัน เช่น `2256000` → `2,256,000`
- `bahtText(numStr)` — แปลงตัวเลขเป็นข้อความจำนวนเงินภาษาไทย เช่น `2256000` → `สองล้านสองแสนห้าหมื่นหกพันบาทถ้วน`
- `stripActingRank(name)` — ตัดคำนำหน้า/ยศรักษาการ (เช่น "ว่าที่ ร.ต.") ออกจากชื่อ
- `xmlEscape(s)` — escape อักขระพิเศษ (`&`, `<`, `>`) ก่อนแทรกลง XML ของ docx
- `normalizePlaceholderRuns(xml)` — เรียกก่อนแทนที่ placeholder ทุกครั้งเสมอ (สำคัญ ดูหัวข้อถัดไป)

## ข้อควรระวัง: Word อาจตัด placeholder ขาดเป็นหลายส่วนในไฟล์ .docx

เวลาพิมพ์ `[ชื่อ_placeholder]` ลงใน Word แล้ว Word บันทึกไฟล์ บางครั้ง Word จะเก็บข้อความนี้แยกเป็นหลาย
"run" (`<w:r>`) ในไฟล์ XML ข้างใน — มักเกิดตรงรอยต่อภาษาไทย/อังกฤษปนกัน (เช่น `[สูตรค่าK]` ที่มีทั้งไทยและ
อังกฤษ) หรือบางทีก็เกิดกับข้อความไทยล้วนๆ อย่าง `[PM1]` ด้วยเหมือนกัน ถ้าไม่จัดการเรื่องนี้ โค้ดแทนที่
placeholder (`replacePlaceholder()`) จะหา `[ชื่อ_placeholder]` ไม่เจอ (เพราะข้อความไม่ได้อยู่ใน `<w:t>`
เดียวกัน) แล้ว placeholder นั้นจะโผล่ค้างเป็นข้อความ `[...]` เดิมๆ ในไฟล์ที่สร้างออกมา ไม่ถูกแทนที่เลย
(เจอบั๊กนี้จริงกับ `[สูตรค่าK]` ในเอกสาร 5 มาแล้ว)

วิธีแก้: เรียก `normalizePlaceholderRuns(xml)` ก่อนเสมอ (ก่อน loop แทนที่ placeholder ทุกตัว) — ฟังก์ชันนี้
จะไล่สแกน run ทีละตัวแล้วรวม run ที่มี "[" ค้างอยู่ (ยังไม่เจอ "]" ปิด) เข้ากับ run ถัดๆ ไปจนกว่าจะเจอ "]"
ให้กลายเป็น run เดียว ใช้ได้กับ placeholder ทุกชื่อโดยไม่ต้อง hardcode รายชื่อ placeholder ไว้ล่วงหน้า —
ถ้าทำเอกสารใหม่แล้วพบว่า placeholder ตัวไหนไม่ถูกแทนที่ (ยังเห็น `[...]` ค้างในไฟล์ที่สร้างออกมา) ให้สงสัย
เรื่องนี้ก่อนเป็นอันดับแรก แล้วเช็คว่าเรียก `normalizePlaceholderRuns()` ก่อน loop แทนที่หรือยัง

## ขั้นตอนเพิ่มเอกสารใหม่ (เช่น เอกสาร 6)

1. สร้างไฟล์ `.docx` ต้นแบบใน `templates/` แล้วพิมพ์ placeholder ที่ต้องการ — **ใช้ชื่อ placeholder ที่มีอยู่แล้วในตารางข้างต้นถ้าความหมายตรงกัน** (เช่น `[ชื่อโครงการ]`, `[เดือน]`, `[ปี]`) เพื่อให้ปุ่ม "สร้างไฟล์ทั้งหมด" ที่หน้า main กรอกให้อัตโนมัติได้เลยโดยไม่ต้องแก้โค้ดเพิ่ม
2. สร้างโฟลเดอร์ `forms/<N>. <ชื่อเอกสาร>/` พร้อมไฟล์ฟอร์มของตัวเองและ `template-data.js` — **ต้อง regenerate base64 จากไฟล์ `.docx` ทุกครั้งที่แก้เทมเพลต** ไม่งั้นแอปจะยังใช้เนื้อหาเก่าอยู่
3. เพิ่ม `TEMPLATE_N_BASE64` ในไฟล์ `forms/main-template-data.js`
4. เพิ่มลิงก์ + ปุ่ม "สร้างไฟล์" ในรายการเอกสารที่ `forms/main.html` และเพิ่มฟังก์ชัน `buildDocN()` (ดูรูปแบบจาก `buildDoc1`-`buildDoc5` ที่มีอยู่)
